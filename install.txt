Overview
--------
A Spitfire BBS-inspired command shell for Synchronet. Recreates
the classic Spitfire look and feel with authentic SF-styled ANSI
menus while using the full Synchronet command set underneath.

Sections: Main / Message / File / Chat / E-Mail/NetMail / QWK


Files Included
--------------
  exec/
    spitfire.js         Command shell

  text/menu/spitfire/
    main.msg            Main menu display
    msg.msg             Message menu display
    file.msg            File menu display
    chat.msg            Chat menu display
    multchat.msg        Multinode chat display
    e-mail.msg          E-Mail/NetMail menu display
    qwk.msg             QWK menu display
    msglist.asc         Message list help display
    msgview.asc         Message view help display

  install.txt           This file
  file_id.diz           Package description


Installation - Windows
----------------------
  1. Copy spitfire.js to \sbbs\exec\

  2. Create the folder \sbbs\text\menu\spitfire\

  3. Copy all .msg and .asc files to \sbbs\text\menu\spitfire\

  4. In SCFG -> Command Shells -> Add new entry:
       Name:          Spitfire Clone
       Internal Code: SPITFIRE


Installation - Linux
--------------------
  1. Copy spitfire.js to /sbbs/exec/

  2. Create the folder /sbbs/text/menu/spitfire/

  3. Copy all .msg and .asc files to /sbbs/text/menu/spitfire/
     Note: filenames must be lowercase (they already are).

  4. In SCFG -> Command Shells -> Add new entry:
       Name:          Spitfire Clone
       Internal Code: SPITFIRE


Notes
-----
  - The Bulletins option (B) requires bullseye.js to be
    present in your exec/ directory.

  - All sections including Chat, E-Mail, QWK, and Multinode
    Chat display Spitfire-styled menus from the spitfire/ folder.

  - Synchronet v3.22a+ automatically falls back to the default
    text/menu/ directory for any menu files not found in the
    spitfire/ subfolder.
